# Claims audit

This is the stable, project-level version of the claims audit. A
run-specific version (checking which policies were actually present in a
given results directory) is generated by:

```bash
python -m helicyn_sim claims-audit --results research_outputs/main_experiment --out research_outputs/claims_audit.md
```

See `helicyn_sim/experiments/claims_audit.py` for the exact logic. The
categories and their meaning:

- **supported**: backed by real evidence (a real trained model, a real
  dataset, or actual simulator output) with a stated caveat.
- **partially_supported**: true in general but depends on run-specific
  conditions (e.g. whether `external_helicyn` was reachable for a given
  run).
- **unsupported**: not backed by anything in this project and must not be
  claimed.

## Supported

- Helicyn ML v1 trains a workload forecaster on real BurstGPT LLM request
  traces. *(evidence: helicyn-ml training pipeline + `python -m helicyn_ml status`)*
- Helicyn ML v1 trains a CPU/memory resource predictor on preprocessed
  Google Cluster traces, `research_usable=yes` for CPU/memory targets.
  *(evidence: `../helicyn-ml/data/processed/resources/google_cpu_memory.parquet`)*
- The simulator implements a reduced-order, CPU/memory-first data-center
  scheduling environment. *(evidence: docs/model_assumptions.md, docs/equations.md)*
- The simulator compares baseline, heuristic, integrated-coordination, and
  (when reachable) external Helicyn policies under documented assumptions.
  *(evidence: `research_outputs/*/aggregate/all_runs_summary.csv`)*
- `integrated_coordination` (a simulator-native, hand-weighted
  coordination-layer heuristic) can be evaluated against baseline in
  simulated scenarios. *(evidence: same as above)*

## Partially supported

- Whether a given research-run/ablation/sensitivity output actually
  includes `external_helicyn` depends on whether `helicyn-ml serve` was
  reachable at `--helicyn-url` when that run happened -- always check the
  specific run's `external_status` before assuming it was included.

## Unsupported

- Production energy savings.
- Verified real-world energy/carbon/cost reductions.
- Real GPU optimization (no GPU-trained model exists anywhere in this project).
- Real cooling optimization (the cooling/PUE model is an analytical proxy).
- A real, validated PUE prediction model.
- A real, validated SLA prediction model (`sla_risk_model` is unavailable/degenerate).
- Validation against real facility telemetry.
- Control of customer or production infrastructure.

See `docs/limitations.md` for the full limitations list this audit is
consistent with.
